## Prerequisites

Before you begin, make sure you have the following installed:

  <Card title="Git" icon="git-alt">
    Download from [git-scm.com](https://git-scm.com). Run `git --version` to verify.

## Set up the project

<Steps>
  <Step title="Clone the repository">
    Clone the project to your local machine:

    ```bash
    git clone https://github.com/namith/3d-portfolio.git
    cd 3d-portfolio
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required packages using your preferred package manager:

    <CodeGroup>

    ```bash npm
    npm install
    ```

    ```bash yarn
    yarn install
    ```

    ```bash pnpm
    pnpm install
    ```

    </CodeGroup>

    This installs React Three Fiber, Drei, GSAP, Tailwind CSS, and all other dependencies listed in `package.json`.
  </Step>

  <Step title="Start the development server">
    Launch the Vite dev server:

    <CodeGroup>

    ```bash npm
    npm run dev
    ```

    ```bash yarn
    yarn dev
    ```

    ```bash pnpm
    pnpm dev
    ```

    </CodeGroup>

    Vite will print a local URL, typically `http://localhost:5173`.
  </Step>

  <Step title="Open the experience">
    Open `http://localhost:5173` in your browser. You will see a loading screen followed by the interactive 3D bedroom scene.

    The scene lighting adjusts automatically based on your system clock. For example, visiting at 9 AM renders the scene with the `park` (morning) environment preset.
  </Step>
</Steps>

## Customize the portfolio iframe

The monitor mesh in the bedroom scene embeds a personal portfolio website inside an `<iframe>`. The URL is set in `src/models/Computer.jsx`:

```jsx src/models/Computer.jsx
<Html
  wrapperClass='monitor'
  position={[-1.93, 1.245, -0.479]}
  transform
  rotation={[1.6, 1.67, -1.6]}
  distanceFactor={0.236}
>
  <iframe
    seamless
    frameBorder={0}
    loading='lazy'
    src="https://namith.vercel.app/"
  />
</Html>
```

Replace `https://namith.vercel.app/` with the URL you want to display on the in-scene monitor.

<Note>
  The iframe loads lazily (`loading='lazy'`). Content only fetches when the component mounts, not on initial page load.
</Note>
